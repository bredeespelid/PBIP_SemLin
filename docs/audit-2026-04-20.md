# PBIP Documenter — Audit 2026-04-20

Datasets tested: `D:\sample_powerbi` (enterprise, 61 tables, BigQuery, 16 FP tables), `D:\Contoso\Contoso\import_contoso_sales.*` (10 tables, 11 measures, 15 visuals, 1 FP, 1 CG), demo (`samples/contoso.json`).

---

## Exec Summary

### D:\sample_powerbi (enterprise, 61 tables, BigQuery)
Works: TMDL parsing, measure extraction, DAX references, doc generation (Markdown/HTML/JSON), lineage engine builds a connected graph, source-trace tab identifies data sources. **Broken**: Switching from this dataset to another leaves stale select options (visual/measure/table selects freeze after first population). Overview diagram stacks many tables and becomes an unusable blob — per-ring radius collapses to a single giant ring, collision passes don't converge at 61 tables. draw.io/Mermaid ERD arrows use wrong cardinality because exporters read `rel.cardinality` (undefined) while the parser produces `fromCardinality`/`toCardinality`. Lineage traces reach the BigQuery data-source node as a label string but project + dataset are discarded — three-part `project.dataset.table` nav chain keeps only the last `Name=` value. No server/database/schema graph nodes. M-step merges parsed but never emitted as graph edges. Calc-column DAX refs ignored by lineage. Impact diagrams have no upstream at all (downstream only).

### D:\Contoso\Contoso\import_contoso_sales.*  (Contoso, 10 tables)
Works: full parse, FP `prm_measures` detected for doc output, calc group `This Year Last Year` parsed with ordinal, lineage fan-out via `modifies_measure` correct. **Broken**: `prm_measures` and `This Year Last Year` tables look identical to ordinary tables in the overview relationship diagram (no FP/CG badge or accent). Detailed ERD shows calc-group brown header but overview SVG has nothing. Switching between Contoso and any other dataset leaves stale lineage SVG (the `_lineageRendered` flag is never reset), stale visual/measure/table select options, and stale field-usage diagrams. Warning banner from a prior parse with errors stays visible on a clean parse. `sponsorBannerClose` listener is re-added every parse, accumulating duplicates.

### Demo mode (samples/contoso.json)
Works: data injected correctly, overview, sidebar, sponsor banner. **Broken**: demo→folder transition does not clear `folderHandle`/`semanticModelHandle`/`reportHandle`. `_lineageRendered` flag not reset before demo data loads, leaving previous lineage SVG on screen. Static cache `MExpressionParser._declaredParams` not nulled, creating cross-dataset coupling.
